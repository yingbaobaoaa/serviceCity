# 智能城市监测系统

基于 Vue 3 + TypeScript + Vite + Element Plus 构建的智能城市监测管理平台。

## 技术栈

- **前端框架**: Vue 3 + TypeScript
- **构建工具**: Vite
- **UI组件库**: Element Plus
- **HTTP客户端**: Axios
- **路由管理**: Vue Router 4
## 开发说明

### 安装依赖
```bash
cd web
npm install
```

### 启动开发服务器
```bash
npm run dev
```

### 构建生产版本
```bash
npm run build
```

### 预览生产构建
```bash
npm run preview
```

city_events.json：市民上报的城市问题（如井盖缺失、占道经营）

4.页面二：问题聚类与热点识别，功能如下
自动识别高频问题类型（如“道路积水”、“噪音扰民”，结合地理区域，发现“问题高发区”
异常关联分析：将传感器异常（如积水>30cm）与附近市民上报事件自动关联
标记“高置信度关联事件”，
用户可选择一条或多条事件/异常，点击“AI分析”，输入结构化数据（含位置、类型、时间、传感器值等）
模拟大模型输出，输出内容包括：
问题归因（如“该区域排水管网老化导致积水”）
处置建议（如“建议调度排水车，并安排管网检修”）
存储历史 AI 分析记录
5.页面三：运行态势大屏，要求独立于前两个页面，全屏展示，具体如下：
使用ECharts创建一个1920x1080的大屏，包含热力图、环形图、折线图等，并模拟实时事件更新,在大屏上定时更新实时事件列表和关键指标；
自定义方法查询数据，数据来源于之前存储的mock下的json数据，自定义方法模拟数据轮询，包含以下至少 5 种可视化组件：

b.问题类型环形图（事件 vs 传感器异常分类对比）
c.实时事件滚动列表（带“NEW”标识，模拟每10秒新增），实时滚动
d.关键指标卡片：今日事件数、异常设备数、高优先级问题数、平均响应时长（模拟）
e.趋势折线图：近7天问题数量变化；

布局建议：顶部指标 + 左侧地图/热力 + 右侧图表 + 底部事件流；
